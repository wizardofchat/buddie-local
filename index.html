§ 0 — Abstract

The Existence Proof

Buddie.local is a live, privately built personal agent system. It started on a Samsung A55 phone running Termux on 25 February 2026 and now runs on a home Mini PC in the UK. Built entirely outside working hours on personal hardware.

It integrates live personal finance, health, and energy data — all stored and processed locally. An Android companion app consumes its REST API. Three AI agents synthesise observations every morning. A structured ensemble deliberates on complex decisions at ~£0.001 per query.

"Known-data queries are answered locally before any LLM is called. Sensitive financial data never leaves the device by architectural design — not by policy promise."

This paper documents the architecture, the engineering decisions, the real operational metrics, and the honest limitations. It is an existence proof: a disciplined local-first architecture can deliver meaningful personal financial intelligence at near-zero cost.

§ 1 — Architecture

Governor, not model. Deterministic first, LLM last.

The central design is a governor layer that routes every query to the cheapest capable tier. Every query passes through five steps: understand intent → audit data freshness → extract surgical context → route to tier → log everything.

§ 2 — Named Agents

Eight agents. Each with a single responsibility.

Every agent runs on a systemd timer, writes to a staging fragment, and reports to the agent_runs table. Each fails independently — one agent failing does not block the others.

Self-extension — the /prompt → /code pipeline

When the system cannot answer a query, it logs the gap to memory/gaps.json. After repeated gaps on the same topic, Buddie proposes a new capability. The pipeline:

/prompt
Natural language specification — describe what the new skill should do, constraints, output format
/code
Code generation — Gemini generates the implementation against VitaClaw schemas and architecture rules
sandbox
Safe execution — runs in isolated context, output reviewed before any promotion
review
Human approval required — generated code is never automatically trusted or promoted
skills/
Permanent capability — promoted to skills/ library, available to all subsequent queries

This has produced functional extensions including energy data fetchers, dividend calendar enrichment, portfolio lens classification, and income stress simulation.

§ 3 — Daily Synthesis

Three agents. Every morning. Observations, not summaries.

At 09:00 daily, three BuddieSynthesis agents run independently. Each receives shaped, anonymised data. Each writes structured observations to ghost_actions. All observations are logged. None are acted on automatically during the 30-day dry run.

08:30
Data refresh — positions, Whoop, energy pulled live from broker and device APIs
09:00
BuddieSynthesis · Finance — portfolio concentration, income pacing, ex-div calendar, DCA effectiveness
09:00

09:00

09:45

Trust architecture

01
No model may both generate and authorize the same action. Observations write to escrow. A separate rule-based verifier stamps them publishable or not.
02
Observations are not recommendations. "What happened?" is automatic. "What should you do?" requires explicit human approval. Financial actions require the strictest gate.
03
Privacy is architectural, not a setting. Tickers anonymised to semantic codes before any cloud call. Health data routes to local-only models. Enforced by the architecture — not a policy statement.
04
Confidence is earned from track record. 30-day ghost run in progress. User ratings on every observation feed a calibration loop. Autonomous actions only unlock when accuracy record earns them.
§ 4 — VitaNest

Kindle-inspired. Intelligence without noise.

VitaNest is a Kotlin/Jetpack Compose Android app consuming VitaClaw's REST API. It follows a Kindle-inspired minimalist design language — cream backgrounds, near-black Noto Serif typography, no colour except where it carries meaning. No animations, no notifications badges, no engagement mechanics. Designed to inform, not to retain.

VitaNest contains zero business logic. No calculations, no hardcoded values, no data storage. VitaClaw is the brain. VitaNest is the window. The relationship is bidirectional: VitaNest reads live data from the API, and writes observation ratings back — feeding the confidence calibration loop.

Portfolio lens — risk classification

Every holding is classified across four dimensions: currency exposure (GBP/USD/mixed), geography (UK/US/global/mixed), income type (dividend/growth/blend), and asset class (ETF/equity/trust/bond). The portfolio lens screen surfaces concentration risk, currency drag, and income distribution at a glance — without any manual tagging.

Health charts — threshold alerting

SpO2 charts display two threshold lines: 92% warning (amber) and 90% critical (red). Breaches surface as alert banners with exact dates and values. The 14-day and all-time toggle changes the chart range with a single tap. 30-day baselines anchor each metric so trends are contextualised against personal history, not population averages.

§ 6 — The Opportunity

Why local-first personal finance intelligence matters.

India has over 100 million retail investors. The UK has 11 million ISA holders. Every one of them shares their most sensitive financial data with cloud platforms they do not own, cannot audit, and pay monthly for.

The personal data model is broken for retail investors.

Cloud AI platforms monetise your financial data, forget your decision history, and charge you for the privilege. The session ends. The system forgets you.

Buddie.local inverts this: your data stays on your device, the system builds permanent memory of your financial history, and the running cost is near-zero. This is not a feature difference. It is an architectural difference.

The architecture is broker-agnostic. The same pattern works with Zerodha, Groww, ICICI Direct, Hargreaves Lansdown, AJ Bell — any broker with an API. A retail investor in Mumbai, Bangalore, or London could run this on hardware they already own.

India's retail investor base is young, mobile-first, and cost-conscious. A system providing institutional-grade portfolio intelligence privately, locally, at near-zero cost addresses a gap that no current product fills at this price point.

§ 9 — Honest Limitations

What this is not. What still needs building.

A credible architecture document acknowledges its boundaries clearly.

Current limitations

Single-user personal system. Not productised, multi-tenant, or production-hardened for regulated financial services.
Self-extension pipeline requires human review before promotion. Generated code is never automatically trusted.
Confidence calibration is in early stage — 30-day dry run in progress. Autonomous actions not yet enabled.
No real-time market data feed — positions refresh daily from broker API, not tick-by-tick.
The capability progress framework is self-defined. It measures milestones, not an industry benchmark.
FCA regulatory boundary: this system produces observations and decision support, not regulated financial advice.
GitHub repository currently private. Key architectural modules will be published to support independent review.
